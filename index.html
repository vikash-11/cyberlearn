Networking Fundamentals
OSI Model — 7 Layers
MASTERED
The 7-layer framework standardizing how data moves across networks — physical cables to app protocols.
networkingfundamentalsprotocols

The 7 Layers (Top → Bottom)

  • 7 — Application: HTTP, DNS, FTP — user-facing protocols
  • 6 — Presentation: Encoding, encryption, SSL/TLS
  • 5 — Session: Opens/manages sessions between apps
  • 4 — Transport: TCP / UDP — port numbers live here
  • 3 — Network: IP addressing + routing (routers live here)
  • 2 — Data Link: MAC addresses (switches live here)
  • 1 — Physical: Raw bits — cables, WiFi signals

Mnemonic

All People Seem To Need Data Processing (7 → 1)

Security Relevance

Each attack targets a specific layer: ARP spoofing = L2, IP spoofing = L3, MITM = L4-7. Know the layer → know the defense.

TCP/IP Model
MASTERED
The real-world 4-layer model powering the internet. Maps loosely to OSI but is what's actually used.
TCPUDPprotocols

4 Layers

  • Application — HTTP, DNS, SMTP, FTP
  • Transport — TCP, UDP
  • Internet — IP, ICMP, ARP
  • Network Access — Ethernet, WiFi (802.11)

TCP vs UDP

TCP — 3-way handshake (SYN→SYN-ACK→ACK). Reliable, ordered. HTTP, SSH, FTP.

UDP — No handshake. Fast, lossy. DNS, VoIP, video streaming.

MAC Addresses & ARP
MASTERED
Hardware identifiers on NICs. ARP resolves IP→MAC on local networks. ARP spoofing = classic MITM setup.
Layer 2ARPMITM

ARP Process

Device broadcasts "Who has IP X?" → Owner replies with MAC → Cache saved locally.

ARP Spoofing Attack

Attacker sends fake ARP replies → poisons victim's ARP cache → all traffic routes through attacker = MITM.

Key Commands

arp -a — view ARP table

arpspoof -i eth0 -t [victim] [gateway] — poison ARP

DNS — Domain Name System
MASTERED
Translates domain names → IP addresses. Runs on UDP port 53. Critical attack surface for spoofing and tunneling.
DNSport 53UDP

Resolution Chain

  • Browser → Local resolver → Root DNS → TLD (.com) → Authoritative
  • Response cached at each step (TTL controls cache lifetime)

Attack Types

DNS Spoofing — injecting fake records to redirect traffic

DNS Tunneling — encoding data in DNS queries for covert exfil

nslookup google.com — basic DNS lookup

Wireless Security
Monitor Mode & Airmon-ng
MASTERED
Switching a WiFi adapter to promiscuous monitor mode to capture all wireless frames — foundation of WiFi auditing.
Kaliairmon-ng802.11

Key Commands

airmon-ng check kill — kill processes that interfere

airmon-ng start wlan0 — creates wlan0mon

airodump-ng wlan0mon — scan all nearby APs

airodump-ng -c 6 --bssid [MAC] -w cap wlan0mon — capture target AP

Managed vs Monitor

Managed — only receives frames addressed to your MAC.

Monitor — captures ALL frames regardless of destination.

802.11 Frame Types
IN PROGRESS
Management, Control, and Data frames. Understanding these is essential for Wireshark analysis and deauth attacks.
802.11framesWireshark

Frame Types

  • Management (Type 0): Beacon, Probe Req/Resp, Auth, Assoc, Deauth
  • Control (Type 1): ACK, RTS, CTS — flow control
  • Data (Type 2): Actual payload frames

Security Relevance

Deauth attacks forge Management frames. WPA2 handshake lives in Data frames. Wireshark filter: wlan.fc.type == 0

nmapreconport scan

Why It Matters

Security data (IDS alerts, Zeek logs, firewall exports) comes as CSV/JSON. Pandas lets you parse, filter and feed it to ML models.

Supervised vs Unsupervised ML
IN PROGRESS
Core ML paradigms and exactly where each one fits in cybersecurity applications.
ML basicsclassificationclustering

Supervised (labeled data)

Train on known malicious/benign samples. Security use cases: spam detection, malware classification, phishing URL detection, intrusion detection.

Unsupervised (no labels)

Find patterns without prior examples. Security use cases: network anomaly detection, zero-day discovery, user behavior clustering (UEBA).

Key Algorithms

Supervised: Random Forest, SVM, XGBoost

Unsupervised: K-Means, DBSCAN, Isolation Forest

Applied Projects
Phishing URL Detector
BUILDING
Build a Random Forest classifier that detects phishing URLs using URL feature engineering and the PhishTank dataset.
projectscikit-learnclassification

Features to Extract from URLs

  • URL length, dot count, use of @ symbol
  • Presence of IP address instead of domain
  • HTTPS or HTTP? Subdomain depth
  • Levenshtein distance from known brands
  • Domain age via WHOIS API

Pipeline

PhishTank dataset → Feature extraction → Train/test split → RandomForestClassifier → Evaluate with precision, recall, F1

Target Metrics

Precision >95%, Recall >90% (minimize false negatives — missing phishing is worse)

Bug Bounty Workflow
LEARNING
The standard 5-step methodology from reading scope rules to writing a clear, impactful report.
methodologyreconreporting

The 5 Steps

  • 1. Scope: Read program rules — know in/out-of-scope targets
  • 2. Recon: Subdomains, ports, tech stack fingerprinting
  • 3. Discovery: Manual testing + automated scanning
  • 4. Exploitation: Prove impact with PoC (screenshot/video)
  • 5. Report: Title, steps to reproduce, CVSS score, fix recommendation

OWASP Top 10
SQL Injection (SQLi)
LEARNING
Injecting malicious SQL into inputs to read, modify or delete database contents. OWASP A03:2021.
OWASP A03injectionSQLmap

Quick Test

Input: ' OR '1'='1 → if login bypassed → vulnerable!

Types

  • In-band: Error-based, Union-based (results in response)
  • Blind: Boolean-based (AND 1=1), Time-based (SLEEP(5))

Tool

sqlmap -u "http://target.com/?id=1" --dbs

XSS — Cross-Site Scripting
LEARNING
Injecting client-side scripts into pages viewed by other users. One of the most common bug bounty findings.
OWASP A03XSSJavaScript

Three Types

  • Reflected: Payload in URL, reflected back in response — requires victim to click link
  • Stored: Script saved in DB, executes for every visitor — highest impact
  • DOM-based: Vulnerability in client-side JS processing

Basic PoC

<script>alert(document.cookie)</script>

Impact

Cookie/session theft, defacement, credential phishing, keylogging.
